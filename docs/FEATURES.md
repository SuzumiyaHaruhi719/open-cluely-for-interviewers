# 功能说明 · Feature Guide

> 面向**面试官用户**和**新加入的贡献者**的功能指南。用大白话讲清楚每个功能：**这是什么 → 为什么有它 → 怎么用 → 背后简单原理**。本文只描述当前真正做出来、能跑起来的功能。

---

## 这个产品一句话是什么

这是一个**面试官的实时副驾（copilot）**。面试进行时，它一边「听」面试官和候选人的对话，一边帮面试官做两件事：

1. **生成追问**——根据候选人刚说的内容，建议下一个该问的好问题；
2. **评估候选人**——随面试推进梳理候选人的能力覆盖情况，结束时给出一份完整评估报告。

注意：它是**站在面试官这一边**的工具，目标是帮你问得更深、评得更准，**不是替候选人答题**。

### 整体怎么跑一场面试（30 秒版）

1. 打开应用 → 弹出选择器，选「线上 / 线下」面试，可选一个样本对话直接试用；
2. 进入一场**只存在内存里**的面试（不保存历史）；
3. 打开麦克风 / 电脑音频开始录音，左侧实时显示转写；
4. 副驾按你的设置**自动追问**，你也可以随时手动点「Generate Q」；
5. 右栏的「Session context」面板随对话更新，显示候选人覆盖了哪些能力、还有哪些缺口；
6. 面试结束点顶栏「总结面试」，得到一份完整评估报告；
7. 想再来一场就点「New interview」，一切清空重来。

## 1. 全新开始，不堆历史（Ephemeral interviews）

**这是什么**
每次打开应用都是一场**全新的、只存在内存里的面试**。没有「最近的面试」列表，关掉或刷新页面，这场面试的转写、追问、总结就全部清空了。

**为什么有它（前因）**
面试内容很敏感（候选人原话、简历、评价）。早期版本会把会话存下来、在侧边栏堆一长串历史，既有隐私顾虑，又让界面越用越乱。所以现在干脆**不做持久化**：
- 隐私——对话不落盘，关掉即清空；
- 简单——侧边栏只有「Live copilot · Question bank · Settings」三项，没有历史列表要管理；
- 干净——每场面试互不串味，上一场的上下文绝不会漏到下一场。

**怎么用**
- 打开应用就会自动弹出「开始新面试」选择器；
- 想重新开始：点侧边栏顶部的「**New interview**」，所有实时状态清空，选择器再次弹出。

**背后简单原理**
所有面试状态都只活在前端 React 的内存里。「New interview」会重置转写、追问结果、右栏面板和总结报告，并通知服务端丢弃这条连接累积的对话缓冲，等于开了一张白纸。

## 2. 线上 vs 线下（Online / Offline）

**这是什么**
开始新面试时要先选一种形式，它决定了**声音怎么采集、谁是谁怎么区分**：

- **线上面试（Online）**——用于远程面试。采集**两条独立声道**：电脑音频（= 候选人的声音）+ 你的麦克风（= 面试官的声音）。因为来源天然分开，谁说的话一清二楚。
- **线下面试（Offline）**——用于现场面试。只用**一只房间麦克风**录下所有人，再靠「说话人分离」把不同人区分开。

**为什么有它（前因）**
两种场景的物理条件完全不同：远程时双方各用各的设备，声音本来就分两路；现场时大家围着一张桌子，只有一只麦克风，必须额外做分离。把这两种情况显式分开，副驾才能用对的方式理解「谁在说话」。

**怎么用**
- 远程视频面试 → 选「线上」，分别开启「Candidate · 电脑音频」和「You · 麦克风」两张卡片；
- 现场面对面 → 选「线下」，界面只显示一张「房间麦克风」卡片。

**背后简单原理**
线上模式下两条声道分别送去转写，候选人走 display 通道、面试官走 mic 通道。线下模式下只有一只麦克风，服务端会额外加上说话人分离来标注「谁说的」（见下一节）。

## 3. 实时转写与说话人分离（Live transcript & speaker roles）

**这是什么**
面试一开口，左侧主栏就实时把语音转成文字，候选人和面试官用不同颜色的气泡区分。

**支持哪些转写引擎（ASR）**
在 Settings 里可选三种语音转文字引擎：
- **DashScope Paraformer**（默认，推荐）；
- **科大讯飞 RTASR（Xunfei）**；
- **豆包 / 火山引擎 Doubao 流式**（需要填你自己的 APP ID + Access Token）。

**线下怎么分辨谁是谁**
- 用**讯飞**时，靠它**自带的角色分离**能力把声音切成不同说话人簇；
- 用其他引擎（如 Paraformer）做线下时，则用本地的 **CAM++** 给每句话标一个说话人编号。

**关键设计：说话人不自动猜「面试官 / 候选人」，而是你点一下指派**
线下转写出来的说话人，默认显示成「**说话人 1 / 说话人 2**」，每条气泡上有「面试官 / 候选人」两个小按钮，**由你点一下来指派身份**。

**为什么这么设计（前因）**
面试一开场常常互相寒暄、抢话、插话，机器这时候去猜「先开口的是面试官」很容易猜错，一错就连累后面所有判断。所以在讯飞角色分离模式下，系统**不替你猜**——把最准的一下交给你：你点一次，这个说话人过去和将来的气泡就都正确归位了，整场只需点一次。

> 小差异：讯飞模式下每个说话人都要你**各自指派一次**（它可能在快速插话时切出超过两个簇）；而 CAM++ 模式下系统会做一个「先开口=面试官」的初始猜测，你纠正其中一个，另一个会自动互补。

**上游断线 / 未就绪怎么办（健壮性）**
- 转写引擎临时出错时，界面会显示一条友好的提示（如「Doubao unavailable: 请在设置里填 APP ID + Access Token」），而**不会让整个应用崩溃**；
- 线下分离还没给某句话标好说话人之前，房间麦克风的原始文字**照常显示**，绝不留白，等分离结果回来再补上身份标签。

**怎么用**
- 转写是自动的，开始录音即出现；
- 线下看到「说话人 N」时，点气泡上的「面试官 / 候选人」即可指派。

**背后简单原理**
浏览器把音频降采样成 16kHz 送给服务端，服务端按所选引擎转写后把文字推回前端。说话人编号 → 角色的映射保存在服务端；你点指派时会同步给服务端，让它对后续句子也用正确身份（这也影响「谁是候选人」的判断，进而影响自动追问）。

## 4. 自动追问（Auto question generation）

**这是什么**
录音过程中，副驾会**自动**在合适的时机生成一条追问，省得你自己一直惦记着「下一个问什么」。追问卡片会带一个小小的「自动」标记。

**两种模式**
在 Settings 里可选自动追问的触发方式：
- **AI 智能追问（agent）**——由一个轻量 AI 监控对话，判断「候选人刚讲完一个值得深挖的点」时才出手，闲聊、半句话、客套都不会触发；
- **每 N 秒自动（interval）**——按固定节奏定时生成，间隔可选 **15 / 30 / 45 / 60 / 90 秒**；这种模式下界面会显示「下次自动追问 ~Ns」的倒计时。

**关键设计：只追问「上次追问之后」的新对话，不重复**
无论哪种模式，每次都只基于**自上一条追问以来的新内容**来生成，所以副驾不会反复追问同一个已经问过的点。

**为什么有它（前因 → 后果）**
- 前因：好的追问需要时机，面试官一边听一边想下一题很累，也容易漏掉值得深挖的点；
- 后果：把「什么时候问、问什么」交给副驾兜底，你可以更专注地听候选人讲；同时「只看新内容」的设计保证它不会变成复读机。

**怎么用**
- 顶栏有个「**Auto**」小按钮，一键开 / 关自动追问；
- 在 Settings 里切换「智能 / 定时」和定时间隔；
- **只有在录音时才会触发**——麦克风一关，自动追问立刻停。

**背后简单原理**
服务端为每条连接跑一个「触发监控」。智能模式用一组**便宜的本地闸门**（冷却时间到了没、有没有足够的新内容、有没有短暂停顿）先过滤，通过后才花一次轻量模型调用问「现在适合追问吗」，任何异常都按「不追问」处理，绝不打断转写。定时模式则用固定时钟节拍触发；两种模式都和手动追问共享同一套「同时只生成一个」的记账，互不打架。

## 5. 手动追问（Generate Q）

**这是什么**
顶栏的「**Generate Q**」按钮，一键基于候选人**最近的回答**立刻生成一条追问。

**为什么有它**
自动追问是兜底，但很多时候你心里清楚「就是现在，我要追这一句」。手动按钮把控制权交回你手上，随时可用。

**怎么用**
- 点顶栏「Generate Q」即可；
- 候选人一旦有新的回答，按钮就会变为可点；
- **线下单麦也能用**——即使还没给任何说话人指派身份，它也会拿房间麦克风的最新文字来生成，不会因为「还没分清谁是谁」而失效。

**背后简单原理**
前端把候选人最近的回答喂进一个分析缓冲区，点击后发给服务端跑同一条追问生成流程（和自动追问完全相同的链路，只是标记为「手动」）。结果以一张丰富的「AI follow-up」卡片显示，包含主问题、备选问题、为什么问、预期能挖到什么证据；专家模式下还会给出多个打分排序的候选问题，你可以一键「选用」其中一个。

## 6. 实时 Session Context 面板（右栏）

**这是什么**
右栏底部的「**Session context**」面板，随面试推进实时显示候选人的评估全景：
- **能力维度（Competencies）**——每项标注状态：**已覆盖 / 部分 / 待探究（缺口）**，用不同颜色区分；
- **已追问主题（Topics）**——已经深入聊过的话题；
- **待探究（Open gaps）**——还没问到、值得补问的缺口。

**为什么有它（前因 → 后果）**
- 前因：这个面板早期是**坏的**——它从老版本界面搬过来时没接上数据，所以一直空着、没有任何反应；
- 后果：现在它真的活了。服务端用一个**轻量模型**每隔几秒分析一次最新转写，把结果实时更新到面板上，让你在面试中随时知道「哪些能力已经验证、哪里还是空白」，从而决定接下来往哪问。

**怎么用**
- 无需操作，开始录音后面板会自动更新；
- 第一次分析出来之前显示一个空状态提示；
- 「New interview」会把它清回空白。

**背后简单原理**
每当有一句话最终确定（候选人或面试官都算），服务端就（去抖后）触发一次轻量模型分析，让它读最近的对话（外加你填的简历 / JD 做参照）输出严格的 JSON：能力维度 + 主题 + 缺口。只在录音中运行，且会避开正在跑的重型追问，免得两个任务抢资源。任何解析失败都会保留上一份好结果，面板绝不显示乱码。

## 7. 面试总结（Interview summary）

**这是什么**
面试结束（或进行中）点顶栏「**总结面试 · Summarize**」，弹出一个窗口，生成一份**完整的候选人评估报告**。报告固定包含这几节：
- **候选人概况**；
- **各能力维度**（每项 1–5 分评分 + 来自面试记录的具体证据）；
- **亮点**；
- **风险 · 不足**；
- **追问覆盖度**（哪些关键点问透了、哪些方向还没探究）；
- **录用建议**（强烈推荐 / 推荐 / 待定 / 不推荐 + 理由）。

**为什么有它**
面试结束后凭记忆写评价，既费时又容易遗漏、带偏见。这份报告**只依据实际对话记录**下结论（证据不足时会明说「证据不足」），给你一个客观、有据可循的评估底稿。

**怎么用**
- 点「总结面试」→ 窗口打开并开始生成，先显示一个加载动画；
- 报告出来后可一键「**复制**」到剪贴板，或「**重新生成**」再跑一份；
- 报告是只读的、安全渲染的（不会被模型输出注入任何标记）。

**背后简单原理**
服务端用 **DeepSeek v4 pro**（最深的推理模型）读取整场对话（双方发言 + 副驾问过的追问都在内）外加 JD / 简历，按上面固定的中文结构产出 Markdown 报告。万一所选模型暂时不可用，会自动回退到备用模型并在报告顶部标注一句，保证你总能拿到一份（可能略浅的）报告。

## 8. 浅色 / 深色主题（Light / Dark theme）

**这是什么**
整套界面采用一套**绿色品牌设计系统（GLP）**，支持浅色和深色两种主题。

**为什么有它**
面试可能在各种光线环境下进行；统一的设计令牌（颜色、间距、圆角等）也让界面在两种主题下都保持一致、好看。

**怎么用**
- 顶栏有一个**太阳 / 月亮图标**，一键在浅色 / 深色之间切换；
- 你的选择会被**记住**，下次打开沿用。

**背后简单原理**
主题通过给页面根元素设置 `data-theme` 切换，深色只覆盖颜色相关的令牌，其余（字体、间距等）两套共用。偏好保存在浏览器本地存储里，刷新后依然生效。

## 9. 辅助能力（简述）

这些是围绕主流程的辅助功能，按需使用：

- **简历 / JD（右栏）**——可以**拖拽上传简历文件**（自动抽取文字）或直接粘贴，也可粘贴岗位描述 JD。填了之后，副驾的追问、右栏面板和总结报告都会参考它们，问得更贴合岗位。
- **Ask AI**——顶栏按钮，基于当前对话向 AI 问一个自由问题（比如「帮我总结目前的对话并建议一个有力的追问」），回答显示在一个独立的结果面板里。
- **Meeting notes / Insights**——顶栏「更多」菜单里，把当前转写整理成会议纪要或面试洞察。
- **简历问答（右栏）**——针对上传的简历做问答，帮你快速了解候选人背景。
- **Question bank（题库）**——侧边栏可切换到题库视图，**浏览**或**语义搜索**真实面试题，按公司、难度筛选。注意：在生成追问时，系统也会从题库里自动检索相关问题作为参考来源，无需你手动操作。

> 顶栏的「截图 / Screen AI」按钮目前是**禁用占位**（标注 Coming soon），图像相关能力尚未实现。

## 一场面试的完整流程（把上面串起来）

1. **打开** → 弹出「开始新面试」选择器。选**线上**（远程，电脑音频 + 你的麦克风）或**线下**（现场，一只房间麦克风）。想先体验可以选一个样本对话（内置「ZH·资深后端/分布式系统工程师面试」和「EN·Senior PM · B2B SaaS Interview」两个），它会预填简历 / JD + 一段对话，方便直接试「Generate Q」。
2. **（可选）填背景** → 右栏拖上简历、粘上 JD，让副驾更懂这个岗位和候选人。
3. **开始录音** → 打开音频卡片，左侧主栏实时出现转写。线下若看到「说话人 1 / 2」，点一下「面试官 / 候选人」把身份定准。
4. **边听边追问** → 顶栏开「Auto」让副驾按你设的「智能 / 定时」节奏自动追问；想自己掌控就随时点「Generate Q」。追问卡片给出主问题、备选、为什么问、预期证据。
5. **看全景** → 右栏「Session context」面板每隔几秒更新，告诉你哪些能力已覆盖、哪里还是缺口，指引你接下来往哪问。
6. **收尾总结** → 点「总结面试」，DeepSeek v4 pro 产出一份带评分、证据、亮点、风险、追问覆盖度和录用建议的完整报告，可复制或重新生成。
7. **下一场** → 点「New interview」，一切清空，回到第 1 步。整场对话不留痕迹，关掉即清空。
